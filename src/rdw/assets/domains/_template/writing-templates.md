# Writing Templates (Template Domain)

## Summary

**Goal:** 2–4 sentences, highest-signal facts only.  
**Structure:** Context → core claim → caveat if needed.  
**Avoid:** Throat-clearing, "In conclusion."

## Analysis

**Goal:** Explain why something is true in domain terms.  
**Structure:**
1. Setup (entity, period, role)
2. Evidence (metrics/facts)
3. Mechanism (why the evidence matters)
4. Limits (what would change the read)

## Comparison

**Goal:** Contrast two entities on named dimensions.  
**Structure:**
1. Shared context
2. Dimension-by-dimension (only dimensions with evidence)
3. Non-claims (what comparison does not show)

## Explainer

**Goal:** Teach a concept using packet facts as illustration.  
**Structure:**
1. Plain definition
2. Domain-precise definition
3. Example from packet
4. Common misuse

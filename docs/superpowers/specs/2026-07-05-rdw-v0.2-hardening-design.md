# RDW v0.2 Hardening — Design

- **Date:** 2026-07-05
- **Status:** Approved (pending spec review)
- **Target release:** v0.2.0 (v0.1.0 ships to PyPI first, unchanged)
- **Branch:** `feat/v0.2-hardening` (off `main`)

## Context

RDW 0.1 is a release-candidate, agent-first CLI harness: it validates research
packets/batches, plans deterministic writing runs, emits prompt bundles, and
installs agent skills. A code review produced a 10-item improvement backlog.

Decisions from brainstorming:

- **v0.1.0 ships to PyPI as-is.** This work targets **v0.2.0** as post-release
  hardening, because #1 and #2 change observable behavior.
- **Wave 3 (#8 lifecycle/state, #9 schema export, #10 provider adapters) is
  deferred.** Each gets its own brainstorm + spec later. #10 in particular stays
  a documented non-goal until a lifecycle state machine (#8) exists — the
  "not an LLM runner" boundary is a deliberate strength.

This spec covers **7 items across 3 waves.** The waves are ordered so the
safety net lands before any behavior refactor.

## Goals

1. Make deterministic router/validation behavior configurable and stricter
   without breaking shipped fixtures or existing tests.
2. Guard the refactors with golden-file snapshots and CI before touching
   behavior.
3. Make file-writing and install operations safe by default (no silent clobber
   or silent directory deletion).
4. Remove version duplication.

## Non-goals

- No lifecycle/state commands, schema export, or provider adapters (Wave 3).
- No new domain packs (legal/finance stay `enabled: false`).
- No change to the "CLI does not call LLM APIs / does not browse" boundary.
- No edits to shipped packet fixtures (the validation design is chosen so they
  keep passing).

## Verified preconditions (from current source)

- Version is duplicated: `__version__ = "0.1.0"` in `src/rdw/__init__.py` and
  `version = "0.1.0"` in `pyproject.toml`.
- No `.github/` exists. 12 tests pass; they assert **substrings**, not
  byte-equal snapshots.
- `infer_contract` and `render_prompt_bundle` are pure/deterministic;
  timestamps live only in `status.json`, not in the contract or bundle.
- `plan_batch` already validates `output_format` and writes it to summary rows,
  but `TaskRequest` / `infer_contract` / the prompt bundle do not carry it.
- `_enabled_or_known_domains()` returns every domain key, ignoring the
  `enabled:` flag; `legal` and `finance` are `enabled: false`.
- Strict validation already enforces source→fact linkage and validates
  `last_updated` as a date; it does **not** validate `accessed`, does not
  require tz-aware timestamps, and only warns when key_facts lack ids.
- `config/router-inference.yaml` already encodes domain/output/audience/entity/
  depth rules, but `planner.py` reimplements them in hard-coded Python
  (`_infer_domain`, `_infer_output`, `_infer_entity`, `_infer_depth`,
  `_infer_audience`).
- **All shipped packets use `source: "Synthetic RDW demo data"`** (free text,
  not URL/DOI). `accessed` values are valid ISO dates; `last_updated` values are
  tz-aware (`...Z`); key_facts are mappings with `id`. Therefore stricter rules
  for accessed/tz/mapping-id are already satisfied, and source-format validation
  must not reject free-text sources.

## Wave 0 — Safety net + hygiene

### #7 Version single-source

`pyproject.toml` remains the single source of truth. Rewrite
`src/rdw/__init__.py`:

```python
from importlib.metadata import PackageNotFoundError, version

try:
    __version__ = version("research-domain-writing")
except PackageNotFoundError:
    __version__ = "0.0.0+local"
```

Update the two tests that hard-code `"rdw 0.1.0"` to assert against the imported
`__version__` (or regex `rdw \d+\.\d+\.\d+`) so a version bump can never break
tests. The v0.2.0 bump then happens in `pyproject.toml` only.

### #6 Golden snapshots + CI

**Golden snapshots.** Add `tests/golden/<slug>/task-contract.yaml` and
`prompt-bundle.md` for the 4 representative requests already exercised in the
suite:

- `improve the copy on my LIS leaderboard`
- `short album blurb about production`
- `explain the API feature for backend engineers`
- `explain idempotency keys`

A new test renders each via `plan_task` and compares **byte-for-byte** against
the golden. `status.json` is excluded (contains a timestamp). Regeneration:
`RDW_UPDATE_GOLDEN=1 uv run pytest` rewrites the goldens. This is the guard for
#1/#2/#4/#5.

**CI.** Add `.github/workflows/ci.yml`, triggered on push + PR, matrix
py3.12/3.13, using `astral-sh/setup-uv`:

```
uv run ruff check .
uv run ruff format --check .
uv run basedpyright src tests scripts
uv run pytest -q
uv build
# wheel smoke:
python -m venv /tmp/rdw-wheel-smoke
/tmp/rdw-wheel-smoke/bin/pip install dist/*.whl
/tmp/rdw-wheel-smoke/bin/rdw doctor
```

## Wave 1 — Correctness quick wins

### #3 Respect `enabled:` / disabled domains

Split domain lookup in `validation.py`:

- `_known_domains(root)` → all keys in `domains.yaml`.
- `_enabled_domains(root)` → keys where `enabled` is not `false` (absent flag
  defaults to enabled).

Behavior in `validate_packet` (new `allow_disabled: bool = False` param):

| Domain state | Non-strict | Strict | Strict + `--allow-disabled-domain` |
| --- | --- | --- | --- |
| Enabled | pass | pass | pass |
| Registered but disabled | **warn** | **error** | pass |
| Not registered | error | error | error |

Add `--allow-disabled-domain` to the `validate-packet` CLI; thread it through.
Tests: a `domain: legal` packet warns non-strict, errors strict, passes with the
flag.

### #4 Propagate `output_format`

- Add `output_format: str | None = None` to `TaskRequest`.
- `infer_contract` emits `output_format` into the contract: use the explicit
  value, else `output-formats.yaml`'s `default_format` (`markdown`). Validate
  against known formats (reuse the `_output_formats` loader); unknown → append to
  `warnings` (do not hard-fail the planner).
- `plan_batch` passes its already-computed `output_format` into `TaskRequest`.
- Add `--output-format` to `task plan`.
- The field appears in the bundle automatically (contract is dumped there); add
  one explicit Execution-Order line: "Save artifacts in the `output_format`
  above, per `config/output-formats.yaml`." Goldens regenerate.

## Wave 2 — Deterministic core (guarded by Wave 0 goldens)

### #1 Config-driven router

New module `src/rdw/router.py` that loads and evaluates
`config/router-inference.yaml`. Reuse the existing config-loading pattern from
`validation.py` (factor the `_load_config` / `_config_root` helpers into a shared
`src/rdw/config.py` so router and validation share one loader).

Rule evaluation:

- **domain_inference:** first domain whose `keywords` substring-match the
  lowercased request; else `general`.
- **output_type_inference:** ordered list; first entry whose `match` token is a
  case-insensitive substring wins, yielding `output_type` (+ optional
  `entity_type`); else `defaults.output_type` (`summary`), entity_type `entity`.
- **audience_inference:** first matching `match` list → `audience`; else
  `defaults.audience`.
- **entity_inference:** explicit `pattern:` regex with `entity_name_template`
  (`{1}` group substitution) or literal `entity_name`; honor `when_no_stat_name`.
- **depth_inference:** `deep`/`light` keyword lists; else `standard`. Explicit
  `--depth` / normalized depth still wins (unchanged precedence).

**Enrich the YAML to fully cover current hard-coded behavior**, so config becomes
the true source (not a partial mirror):

- Add an idempotency entity rule (`pattern` or keyword) yielding
  `entity_name: "Idempotency keys"`, domain `technical`.
- Ensure `idempotency` maps to `feature_explainer` in `output_type_inference`.
- Audit **every** hard-coded branch for divergence before deleting it. Known
  gap: hard-coded `_infer_output` fires `stat_interpretation` on the bare token
  `"stat"`, but the YAML rule is `stat interpretation` (won't substring-match
  `"stat"`). Add a `stat` keyword (or equivalent) so the substring engine
  reproduces this. Any branch whose YAML equivalent cannot be reproduced 1:1 is
  a deliberate behavior change and must be recorded as an intended golden
  regeneration, not an accident.

Then **delete** `_infer_domain`, `_infer_output`, `_infer_entity`,
`_infer_depth`, `_infer_audience` from `planner.py` (dead-code removal per the
quality ladder). `infer_contract` calls into `router.py`.

Guard: existing `test_router_infers_*` tests plus Wave 0 goldens must stay green
byte-for-byte. Any intended change to inferred output requires a deliberate
golden regeneration in the same commit.

### #2 Stricter `--strict`

Additions to `validate_packet` / `_validate_source_notes`:

1. **key_facts shape (strict):** every item must be a mapping with a non-empty
   `id`. Currently only a non-strict warning. Error:
   `key_facts[i] must be a mapping with an id in strict mode`.
2. **`accessed` date (all modes):** if present and unparseable, error
   `source_notes[i] accessed must be an ISO date`. (Fixtures use valid dates.)
3. **tz-aware `last_updated` (strict):** parse and require `tzinfo`. Error:
   `last_updated must be timezone-aware in strict mode`. (Fixtures use `Z`.)
4. **Source format (strict, pragmatic):** validate the `source` string **only
   when it looks like a URL or DOI** — a malformed `http(s)://…` or `10.…/…`
   errors; free-text / synthetic / interview sources pass unchanged. Recognize
   an optional `source_type` field (`url|doi|book|interview|dataset|synthetic|
   other`); when provided, validate `source` against that type's shape.

This design keeps every shipped packet valid under `--strict` with no fixture
edits (verified against demo-guard + music + technical packets).

### #5 Safe write / install

**`plan_task` (additive; default behavior unchanged):**

- `--force`: explicit overwrite (documents intent; same effect as today's
  default).
- `--no-overwrite`: error if `task-contract.yaml` already exists in the target.
- `--run-id auto|<id>`: write into `<out>/<run-id>/` (`auto` = timestamp slug),
  avoiding clobber entirely.

**Install (`_replace_symlink` and CLI):**

- `--dry-run`: compute and print planned writes/symlinks; touch nothing.
- `--backup`: before replacing a real (non-symlink) directory, move it to
  `<path>.bak-<timestamp>` instead of `shutil.rmtree`.
- `--force`: **gate the destructive `rmtree` behind this flag.** Without
  `--force` or `--backup`, a real directory occupying a skill-link path is an
  error with a clear message, rather than being silently deleted.
- **Windows fallback:** if `os.symlink` raises `OSError` (no privilege), fall
  back to `copytree` and note in output that a copy (not a link) was created.

Existing install test passes unchanged: it installs into a fresh temp `$HOME`
where no real directory occupies the link paths.

## Testing strategy

- **Golden snapshots** (Wave 0) for all 4 representative requests; regenerated
  intentionally when #4/#1 change output.
- **#3:** disabled-domain packet across non-strict / strict / strict+flag.
- **#4:** `output_format` present in contract + bundle; batch propagation;
  unknown format warns.
- **#1:** existing router assertions stay green; a new test asserts the router
  reads from a `--root` config (config-driven, not hard-coded) by pointing at a
  temp `router-inference.yaml` with an altered keyword.
- **#2:** key_facts-without-id (strict error), bad `accessed` date, naive
  `last_updated` in strict, malformed URL source; and a positive test that all
  three shipped packets still pass `--strict`.
- **#5:** `--no-overwrite` errors on existing contract; `--run-id auto` writes a
  subdir; install `--dry-run` writes nothing; install `--backup` preserves the
  prior directory; symlink-failure fallback copies.

## Quality ladder (per completion)

`uv run ruff check .` · `uv run ruff format --check .` ·
`uv run basedpyright src tests scripts` · `uv run pytest -q` · `uv build` +
wheel smoke. Update `.tracker/PROJECT_TRUTH.md` after each committed wave.

## Rollout / commits

Atomic commits scoped per item (or per wave for the small ones), truth-file
update after each. Suggested order = Wave 0 → 1 → 2. Version bump to `0.2.0` in
`pyproject.toml` happens at release cut, after Wave 2 lands and the ladder is
green.

## Open questions

None blocking. Source-format taxonomy (`source_type`) is intentionally optional
and can expand in a later minor version without breaking packets.

# Concept: Failure-mode-first

**Domain:** technical  
**Slug:** failure-mode-first

## Definition

Technical writing should name how a system fails (latency spikes, partial outages, bad states) before praising benefits — especially for operators and implementers.

## When to use

- Architecture summaries, risk memos, feature explainers for engineers

## When not to use

- Pure marketing one-liners unless user explicitly requests marketing mode

## Writing guidance

- Pair every benefit with constraint, limit, or operational caveat from packet
